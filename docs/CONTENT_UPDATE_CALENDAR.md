# Content Update Calendar

A maintained list outranks an abandoned list.

## Weekly

- Add 3-10 useful platforms or improvements.
- Fix broken links from issues.
- Review open platform submissions.
- Update `CHANGELOG.md`.

## Monthly

- Run `python scripts/check_links.py`.
- Add or improve one long-tail guide.
- Submit one careful backlink/PR to a related list.
- Refresh `data/platforms.json` if README entries changed.

## Quarterly

- Review categories and remove dead/thin platforms.
- Add newly relevant music-tech surfaces.
- Improve comparison tables.
- Update GitHub topics/description if positioning changed.

## Release cadence

Use lightweight tagged releases for major directory expansions:

- `v0.1.0` initial public launch
- `v0.2.0` comparison-table expansion
- `v0.3.0` long-tail guide expansion
- `v1.0.0` stable public directory baseline
