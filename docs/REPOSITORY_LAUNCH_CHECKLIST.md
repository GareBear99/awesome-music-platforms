# Repository Launch Checklist

Use this after dragging the package into GitHub.

## GitHub repository settings

- [ ] Repo name is lowercase: `awesome-music-platforms`.
- [ ] Description is set to the recommended SEO description.
- [ ] Website points to `https://github.com/GareBear99/awesome-audio-lists`.
- [ ] Topics are added from `docs/GITHUB_TOPICS.md`.
- [ ] Issues are enabled.
- [ ] Discussions are enabled if you want community submissions.
- [ ] Sponsorship/FUNDING links are set when ready.
- [ ] Social preview image is uploaded from `assets/social-preview.svg` or exported PNG.

## First public commits

- [ ] Commit README + docs + data + workflows.
- [ ] Create first release: `v0.1.0`.
- [ ] Add repo link to `awesome-audio-lists`.
- [ ] Add repo link to `awesome-audio-plugins-dev` if relevant.
- [ ] Add repo link to TizWildinEntertainmentHUB if relevant.

## First outreach

- [ ] Submit to one relevant awesome list.
- [ ] Post in one music production community with useful context.
- [ ] Add a Facebook post explaining the directory for independent artists.
- [ ] Share as a tool, not an advertisement.
